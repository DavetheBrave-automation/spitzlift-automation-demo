Token Management Rules

How To Not Fry Chrome's Brain — Reference: SpitzLift Chrome session hit 206,408 tokens and timed out

  1. 1
    HARD LIMIT per Chrome session: 20 tool calls. Write this at the top of every Chrome prompt. Without a written limit, Chrome will go until it runs out of tokens or the session dies.
  2. 2
    SUMMARIZE AS YOU GO. Never ask Chrome to "read the full page" — always "extract and summarize only." Full-page reproductions are the single biggest token killer.
  3. 3
    SPLIT BY TOPIC not by time. Each Chrome session owns one topic completely. Don't mix leadership research with competitive research in the same session.
  4. 4
    REPORT FORMAT FIRST. Start every Chrome session with "Keep total output under [X] words" — this forces summarization and prevents token bloat from the first tool call.
  5. 5
    TERMINAL FOR APIS. Anything that can be done via an API (SAM.gov, USASpending, site crawl, revenue data) goes to Terminal. Chrome is for sites that require navigation and interpretation. Never use Chrome for something Terminal can do.
  6. 6
    NO FULL PAGE REPRODUCTIONS. EVER. Snippets and summaries only. If it's important enough to copy in full, save the URL and fetch it separately in a dedicated session.
